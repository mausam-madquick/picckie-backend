# 🌐 VPS Deployment Guide for Background Remover Server

Deploying on a Virtual Private Server (VPS) like DigitalOcean, Linode, AWS EC2, or Hetzner gives you full control and high performance.

Since we've already set up your project with Docker, deployment on your VPS is incredibly clean. Below are the two best methods to deploy your backend:

---

## ⚡ Method 1: Using Docker (Recommended, Easiest)

Docker runs both the Node.js gateway and the Python AI background remover together in a single lightweight container.

### Step 1: Install Docker on your VPS
Log into your VPS via SSH and install Docker (assuming Ubuntu/Debian):
```bash
sudo apt update && sudo apt install -y docker.io
sudo systemctl enable --now docker
```

### Step 2: Clone and Build
Clone your repository and build the Docker image. The build will automatically compile dependencies and pre-download the 176MB AI model so it's ready instantly!
```bash
# Clone the repository
git clone https://github.com/mausam-madquick/picckie-backend.git
cd picckie-backend

# Build the Docker image
sudo docker build -t picckie-backend .
```

### Step 3: Run the Container
Run the container in the background. It will bind the Express gateway to port `8001` on your VPS, and will restart automatically if the server reboots:
```bash
sudo docker run -d \
  -p 8001:8001 \
  --name picckie-backend \
  --restart unless-stopped \
  picckie-backend
```

### Step 4: Verify Deployment
Run the following command to check if the server is healthy:
```bash
curl http://localhost:8001/health
```
You should get `{"status":"ok","service":"image-processor"}` back!

## 🟢 Method 2: Manual Setup (Using PM2 Process Manager)

If you prefer not to use Docker, you can run the two services as standard system processes managed by **PM2**.

### Step 1: Install Node.js, Python, and PM2
On your VPS, install Node.js, Python 3, virtualenv, and PM2:
```bash
# Install Node.js & NPM
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs python3-pip python3-venv

# Install PM2 globally
sudo npm install -g pm2
```

### Step 2: Clone and Setup Python Service
```bash
git clone https://github.com/mausam-madquick/picckie-backend.git
cd picckie-backend

# Go to Python directory and set up virtualenv
cd python-bg-remover
python3 -m venv venv
source venv/bin/activate

# Install requirements
pip install -r requirements.txt

# Start Python service with PM2 in background
pm2 start "uvicorn app:app --host 127.0.0.1 --port 8000" --name "picckie-python-remover"
```

### Step 3: Setup Node.js Express Gateway
```bash
cd ../server
npm install

# Start Express gateway with PM2
# Note: It automatically connects to localhost:8000 where Python is running
pm2 start server.js --name "picckie-node-server" --env PORT=8001
```

### Step 4: Keep Processes Alive After VPS Reboot
```bash
pm2 save
pm2 startup
```
*(Copy and paste the command generated by `pm2 startup` to enable boot scripts).*

## 🛡️ Production Security: Nginx Reverse Proxy & SSL (HTTPS)

For a secure production deployment, you shouldn't expose raw ports like `:8001` to your users. It's best to configure **Nginx** as a reverse proxy and secure it with a free SSL certificate from **Let's Encrypt**.

### Step 1: Install Nginx & Certbot
```bash
sudo apt install -y nginx certbot python3-certbot-nginx
```

### Step 2: Configure Nginx
Create a configuration file for your domain (e.g., `api.picckie.com`):
```bash
sudo nano /etc/nginx/sites-available/picckie-backend
```

Paste the following configuration:
```nginx
server {
    listen 80;
    server_name api.picckie.com; # Replace with your subdomain/domain

    # Set maximum upload size for high-res images
    client_max_body_size 50M;

    location / {
        proxy_pass http://127.0.0.1:8001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

Enable the site and restart Nginx:
```bash
sudo ln -s /etc/nginx/sites-available/picckie-backend /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### Step 3: Get Free SSL (HTTPS) Certificate
Secure your subdomain instantly:
```bash
sudo certbot --nginx -d api.picckie.com
```
*Select option **2** to automatically redirect HTTP requests to secure HTTPS.*

## 🎨 Update your Frontend (`picckie`)
Once deployed on your VPS, configure your frontend `.env` inside `picckie/.env`:

```env
# If using Nginx with domain:
NEXT_PUBLIC_BACKEND_URL=https://api.picckie.com

# If using raw IP without SSL:
NEXT_PUBLIC_BACKEND_URL=http://YOUR_VPS_IP:8001
```
